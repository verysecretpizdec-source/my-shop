# My Shop — Интернет-магазин на Next.js 15

Привет! Это мой первый серьёзный пет-проект — небольшой интернет-магазин.

Я делал его примерно 3–4 недели, начиная с нуля на Next.js 15 (App Router) + TypeScript. Хотел попробовать современный стек, который часто встречается в вакансиях.

## Что умеет магазин
- Каталог из 12 товаров (поиск, фильтры, сортировка)
- Корзина на Zustand (persist в localStorage)
- Оформление заказа через Server Action

## С чем пришлось повозиться
Самое неприятное было с картинками. Сначала пытался через Unsplash — то 404, то ошибки 500 в Next Image. В итоге плюнул и сделал просто через `<img>` + fallback-заглушку, если картинка не грузится. Стало надёжно (и я наконец-то перестал залипать на этой проблеме).

Ещё мучился с пересчётом суммы в корзине — иногда UI моргал. Пришлось сделать небольшой `setTimeout`, чтобы всё ровно обновлялось. Оставил TODO-комментарий, потому что знаю, что потом лучше сделать через `useMemo`. На будущее хочу разобраться и сделать это аккуратнее.

## Технологии

## Как запустить проект

Ничего сверхъестественного. Ставишь зависимости и поднимаешь dev-сервер:

```bash
npm install
npm run dev
```

Открой в браузере (http://localhost:3000). Если что-то падает — сначала глянь версию Node (у меня нормально работало на актуальном LTS).

Иногда полезно прогнать `npm run build` перед деплоем, чтобы поймать ошибки типов. Линт: `npm run lint`, тесты: `npm run test` (там немного unit-тестов, ничего сложного).

## Структура проекта (коротко)

Я не заморачивался с глубокой архитектурой, но папки примерно такие:

- `app/` — страницы, layout, server action на checkout, роут `/api/products` с товарами
- `components/` — в основном один большой `shop-client`, карточка товара, тема, провайдеры
- `lib/` — данные товаров и мелкие утилиты
- `store/` — Zustand-корзина
- `types/` — типы
- `tests/` — пару тестов на корзину и карточку

Если копаться — всё лежит близко к корню, без лишних вложенных папок «на вырост».

## Что я улучшил бы дальше (2–3 пункта)

Нормальный вопрос. Если бы возвращался к проекту:

1. **Корзина и оформление** — вынести часть логики из огромного компонента, может отдельная страница заказа или модалка, чтобы не держать всё в одном файле.
2. **Картинки** — либо статика из `public/`, либо один нормальный CDN без сюрпризов, чтобы вообще не думать про fallback в проде.
3. **Тесты** — добавить пару проверок на фильтры/поиск без фанатизма, чтобы при правках не ломать каталог молча.
